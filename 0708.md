## Recommendations

1. **Add a build pipeline** вЂ” the `velite generate` step must be automatic. Without it, both `next dev` and `next build` fail silently.
2. **t** вЂ” the `velite generate` output is gitignored. The build pipeline on Vercel must run `npx velite generate` before `next build`.
3. **Extract shared layout** вЂ” the footer is duplicated across all 4 pages with near-identical markup. A shared `Footer` component would reduce maintenance.
4. **Add `next.config.ts` Velite integration** вЂ” consider using Velite's Next.js plugin or a custom `generateBuildId` to ensure `.velite` is always fresh.
5. **Review real vs placeholder data** вЂ” before deployment, audit all placeholder values (stats, fake posts, fake projects, fake views/likes).

### 6. Accessibility Basics
- **All images have alt text** вЂ” `alt={post.title}` or `alt={project.title}` on every `Image` component. (Severity: OK)
- **All interactive elements are focusable** вЂ” `<Link>`, `<a>`, `<button>` used throughout. (Severity: OK)

#### Medium
- **`text-ink/50` and `text-ink/60` opacity text** вЂ” `src/app/about/page.tsx:28`, `src/app/blog/page.tsx:52`, and other pages use 50-60% opacity on `#2B2B2B` ink text. On the `#FDF8F0` parchment background, reduced-opacity text fails WCAG AA contrast (3:1 minimum for body text). Example: `text-ink/40` on `blog/page.tsx:139` вЂ” `#2B2B2B` at 40% opacity = `#A8A5A0` on `#FDF8F0` parchment = approximately 2.5:1 contrast ratio, below the 4.5:1 AA threshold for body text. (Severity: Medium)
- **`bg-primary/10` tag backgrounds** вЂ” Primary red `#C23B22` at 10% opacity on `#FDF8F0` parchment creates very low contrast for the tag text, which is then set to `text-primary` (full red). The tag text may be hard to read for users with visual impairments. (Severity: Medium)
- **Font subsets** вЂ” `src/lib/fonts.ts:5,14,19` вЂ” All three Google Fonts use `subsets: ['latin']` only. Chinese characters (CJK) will NOT be loaded. The calligraphy and body fonts will fall back to system fonts on real Chinese text. Ma_Shan_Zheng and ZCOOL_XiaoWei are specifically Chinese fonts вЂ” without the `chinese-simplified` subset, they render nothing for Chinese characters. **This is a critical rendering bug.** (Severity: High)
