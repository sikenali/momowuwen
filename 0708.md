## Recommendations

1. **Add a build pipeline** — the `velite generate` step must be automatic. Without it, both `next dev` and `next build` fail silently.
2. **t** — the `velite generate` output is gitignored. The build pipeline on Vercel must run `npx velite generate` before `next build`.
3. **Extract shared layout** — the footer is duplicated across all 4 pages with near-identical markup. A shared `Footer` component would reduce maintenance.
4. **Add `next.config.ts` Velite integration** — consider using Velite's Next.js plugin or a custom `generateBuildId` to ensure `.velite` is always fresh.
5. **Review real vs placeholder data** — before deployment, audit all placeholder values (stats, fake posts, fake projects, fake views/likes).
